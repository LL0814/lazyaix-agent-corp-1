# Skill：需求对齐（requirement_alignment）

> **技能类型**：决策模式（Decision Pattern）
> **对应代码**：`scripts/handle.py`（本 skill 文件夹内）
> **共享依赖**：`common/`（skills 公共模块，挂载点见智能体配置）

---

## 一、技能概述

当用户提供的旅需信息不完整时，Agent 不立即调用工具，而是通过追问引导用户补全核心槽位（目的地、天数、预算），确保行程规划有足够输入。

## 二、触发条件

- 用户输入经槽位提取后，`destination` / `days` / `budget` 任一为空
- 且当前 memory 中无已生成行程
- 且用户未表达"校验"或"重置"意图

## 三、输入

| 参数 | 类型 | 说明 |
|------|------|------|
| `req` | UserRequirement | 已合并多轮槽位的需求对象 |

## 四、处理逻辑

```
slot_extractor.extract()  ← 提取本轮槽位
   │
   ▼
与 memory 中已有槽位合并  ← 支持多轮累积
   │
   ▼
UserRequirement.is_complete()?
   │
   ├─ 否 → 生成追问文案（指出缺失项 + 复述已识别项）
   │        返回 {"action": "direct", "response": "..."}
   │
   └─ 是 → 不进入本模式，转 itinerary_planning
```

### 槽位提取规则（基于规则匹配，不依赖 LLM）

| 槽位 | 提取方式 | 示例 |
|------|----------|------|
| `destination` | 正则匹配"去/到/前往+地名" + 已知目的地词表 | "想去成都" → 成都 |
| `days` | 阿拉伯/中文数字 + 天/日 | "玩3天" / "五天" → 3 / 5 |
| `budget` | 数字+元/千/万 + "预算+数字" | "预算3000元" → 3000 |
| `budget_level` | 关键词（穷游/舒适/豪华）或按金额推断 | "穷游" → low |
| `preferences` | 偏好关键词词表匹配 | "喜欢美食" → 美食 |

### 多轮合并机制

```
第1轮："我想去成都"    → destination=成都
第2轮："玩3天"        → days=3
第3轮："预算3000元"    → budget=3000 → 齐全 → 触发生成
```

## 五、输出

```python
{
    "action": "direct",
    "response": "已了解：目的地：成都，天数：3 天\n您的预算大概是多少？比如 3000 元、穷游、舒适等。"
}
```

Agent 直接将 `response` 返回给用户，不调用任何 Tool。

## 六、依赖

- **Tools**：无
- **共享模块**：`common.slot_extractor`、`common.models.UserRequirement`

## 七、对话示例

```
用户：我想去成都
Agent：已了解：目的地：成都
       您计划玩几天呢？ 您的预算大概是多少？比如 3000 元、穷游、舒适等。

用户：玩3天
Agent：已了解：目的地：成都，天数：3 天
       您的预算大概是多少？比如 3000 元、穷游、舒适等。

用户：预算3000元
Agent：（槽位齐全，转行程规划模式 → 调用 generate_itinerary 工具）
```

## 八、扩展方向

- 替换规则提取为 LLM 提取（支持更自然的表达）
- 增加槽位校验（如天数合理性、预算与天数匹配）
- 支持目的地别名（如"魔都"→上海）
