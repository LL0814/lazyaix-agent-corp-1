# Skill：行程校验（itinerary_validation）

> **技能类型**：决策模式（Decision Pattern）
> **对应代码**：`scripts/handle.py`（本 skill 文件夹内）
> **校验逻辑**：`common/validator`（skills 公共模块，挂载点见智能体配置）

---

## 一、技能概述

当行程已生成且用户要求校验时，Agent 对行程进行逻辑检查（时间冲突、交通合理性、预算异常等），输出校验报告与修正建议，不调用任何外部 Tool。

## 二、触发条件

以下条件**全部满足**时触发：

1. memory 中存在已生成行程（`current_itinerary is not None`）
2. 用户输入命中校验意图关键词：`校验 / 检查 / 验证 / 有问题吗 / 可行吗 / 看看行程`

## 三、输入

| 参数 | 类型 | 说明 |
|------|------|------|
| `memory` | Memory | 读取 `current_itinerary` |

## 四、处理逻辑

```
用户要求校验 + 已有行程
   │
   ▼
从 memory 读取 Itinerary 对象
   │
   ▼
validator.validate(itinerary)
   │
   ├─ 检查 1：景点开放时间 vs 到达时间
   ├─ 检查 2：相邻景点交通耗时是否合理
   ├─ 检查 3：每日景点数量是否过多/过少
   └─ 检查 4：预算汇总是否异常
   │
   ▼
格式化校验报告（formatter.format_validation_report）
   │
   ▼
若校验通过 → 附带展示完整行程
   │
   ▼
返回 {"action": "direct", "response": 校验报告文本}
```

### 校验规则详情

| 检查项 | 判定标准 | 问题示例 | 建议 |
|--------|----------|----------|------|
| 开放时间 | 到达时间不在 open_time 范围内 | "武侯祠到达时间 07:00 不在 08:00-18:00 内" | 调整游览时间 |
| 交通耗时 | 单段路程 > 4 小时 | "第1段路程耗时 5.0 小时，过长" | 减少景点或换近的 |
| 交通耗时 | 单段路程 > 2 小时（警告级） | "第2段路程耗时 2.5 小时，偏长" | — |
| 景点数量 | 每日 > 5 个 | "安排了 6 个景点，过于紧凑" | 不超过 3-4 个 |
| 景点数量 | 每日 = 0 个 | "没有安排任何景点" | — |
| 预算汇总 | total ≤ 0 | "总预算为 0" | — |
| 日均花费 | daily_avg > 2000 | — | 考虑降低住宿标准 |

## 五、输出

```python
{
    "action": "direct",
    "response": """==================================================
  行程校验报告
==================================================
发现 2 个问题：

  1. 2026-07-01：武侯祠 的到达时间 07:00 不在开放时间 08:00-18:00 内
  2. 2026-07-01：安排了 6 个景点，可能过于紧凑

修正建议：
  1. 建议调整 武侯祠 的游览时间到 08:00-18:00
  2. 建议每日景点不超过 3-4 个，留出休息时间
=================================================="""
}
```

校验通过时，response 中会附带完整行程展示。

## 六、依赖

- **Tools**：无（纯逻辑校验）
- **共享模块**：`common.validator`、`common.formatter`
- **数据结构**：`Itinerary` / `POI` / `Route`（由 tools 模块定义）

## 七、对话示例

```
（行程已生成）
用户：帮我校验一下行程
Agent：==================================================
         行程校验报告
       ==================================================
       行程检查通过，未发现明显问题。

       ==================================================
         成都 旅行行程方案
       ==================================================
       【第 1 天】...
```

## 八、行程恢复机制

Agent 执行 Tool 后，Itinerary 对象会存入 `memory["history"]`。本模式通过 `common/utils.py` 的 `restore_itinerary_from_memory()` 遍历 history，找到最近的 Itinerary 对象缓存到 `current_itinerary`，确保跨轮可用。

## 九、扩展方向
